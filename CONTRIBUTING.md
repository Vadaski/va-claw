# Contributing

Thanks for your interest in va-claw.

## Local development

- `pnpm install`
- `pnpm build`
- `pnpm test`

Run all three before opening a pull request.

## Code standards

- TypeScript strict mode
- ESM modules
- Keep each source file under 200 lines when editing/adding
- Prefer clear CLI output and concise implementation

## Issues

- For bugs, use the bug report issue template and include reproducible steps.
- For feature ideas, use the feature request template.
- If a report is missing required information, we may ask follow-up questions before triage.

## Pull requests

- One PR should focus on one topic.
- Keep the scope small and explicit.
- Link related issues where applicable.
- CI/tests must pass before merge.
- Include any behavior changes in docs when needed.

## Questions

Open an issue if you are unsure about command behavior or expected configuration.
